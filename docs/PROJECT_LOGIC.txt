1. PROJECT OVERVIEW
================================================================================

What it is:
  A Streamlit dashboard for browsing, analyzing, and reading about Polymarket
  prediction markets. It has three modes:
    - Live Newspaper: AI-generated editorial articles about trending markets
    - Market Browser: Browse by category, view odds, use bet calculator
    - All Analytics: Charts, P&L estimates, accuracy stats on resolved markets

Tech stack:
  - Python 3.x + Streamlit 1.54.0
  - Google Gemini API (gemini-2.5-flash, free tier) for article generation
  - Polymarket Gamma API (public, no auth) for market data
  - Plotly for interactive charts
  - Pandas for data processing

How to run:

  C:/Users/noama/projects/polymarket/venv/Scripts/python.exe -m streamlit run dashboard/app.py --server.port 8520

Dependencies (requirements.txt):
  streamlit, pandas, plotly, requests, google-genai

3. ARCHITECTURE & DATA FLOW
================================================================================

Data sources:
  1. Polymarket Gamma Events API -> Live events (newspaper + market browser)
     Endpoint: https://gamma-api.polymarket.com/events
  2. Polymarket CLOB Prices API  -> Price history charts
     Endpoint: https://clob.polymarket.com/prices-history
  3. Local file: data/raw/markets_raw.json -> Analytics (offline dataset)

Flow for NEWSPAPER mode:
  fetch_newspaper_events(limit=15)         # Live API call, sorted by 24h volume
    -> news_events (list of event dicts)
    -> Trim to _events_for_llm             # Serialize key fields for LLM
    -> get_newspaper_articles(json, key)   # Cached Gemini call (15 min TTL)
      -> generate_articles_batch()         # Single batch prompt to Gemini
      -> Returns Dict[event_id -> article_text]
    -> Render: Masthead -> Ticker -> Lead -> Top Stories -> Market Watch -> Hot Takes

Flow for MARKET BROWSER mode:
  get_top_categories(limit=14)             # Scans top 500 events for tag frequencies
    -> Category pills shown as radio buttons
  load_events_for_category(slug)           # Fetches events for selected category
    -> Filter/sort/paginate
    -> Render event cards with expanders
    -> Each market has bet calculator (active) or resolution panel (resolved)

Flow for ANALYTICS mode:
  load_markets()                           # Reads data/raw/markets_raw.json
    -> pd.DataFrame with derived columns (primary_tag, resolved_winner, pnl, etc.)
    -> Renders: KPIs, Top 15 chart, category treemap, timeline, accuracy, upsets

4. MODE 1: LIVE NEWSPAPER ("The Polymarket Chronicle")
================================================================================

Location in app.py: Lines ~1068-1235

FETCHING:
  fetch_newspaper_events(limit=15) in fetch_events.py:
    - Queries Gamma API: order=volume24hr, closed=false, active=true
    - Fetches 4x the limit (60 events), then scores and ranks
    - Scoring formula: (vol_24h * time_score) + (competitive * 100000)
    - Time score multipliers:
        event > 6 months away   -> 0.3x (penalize)
        event < 30 days away    -> 1.5x (boost)
        otherwise               -> 1.0x
    - Filters: vol_24h >= $1K OR vol_total >= $10K
    - Returns top 15 by score

LLM ARTICLE GENERATION:
  For each event, builds _events_for_llm with:
    - id, title, volume, volume24hr, liquidity, tags, competitive
    - markets[]: up to 10 sub-markets for multi-outcome, 1 for binary
  Calls get_newspaper_articles(json_string, api_key, refresh_key)
    - Cached via @st.cache_data(ttl=900) — 15 minute TTL
    - refresh_key parameter allows cache busting (incremented on "Fresh Edition" click)
    - Internally calls generate_articles_batch() -> single Gemini API call
    - Falls back to template articles if no API key or API error

REFRESH MECHANISM:
  st.session_state["np_refresh"] counter
    - Starts at 0
    - "Fresh Edition" button increments it and calls st.rerun()
    - The counter is part of the cache key for get_newspaper_articles()
    - Even if market data is identical, new counter = cache miss = fresh articles

RENDERING SECTIONS:
  1. MASTHEAD
     - "THE POLYMARKET CHRONICLE" title (Playfair Display font)
     - Date, edition number (days since 2025-01-01), "Live Markets" label
     - Gradient text effect (white -> blue)

  2. TICKER BAR
     - Top 8 events, scrollable horizontal strip
     - Shows: title (truncated) + leading outcome + probability
     - Green for high-probability leaders, red for low
     - Entity icons (flags, sports, crypto, people) prepended for recognized outcomes

  3. LEAD STORY (news_events[0])
     - Full-width article with 2-column layout: text + pull-quote sidebar
     - Pull-quote shows top probability as large gradient number
     - Multi-outcome events show outcome grid below article
     - Stats bar: Volume, 24h volume, Liquidity
     - "Explore odds & bet calculator" button -> navigates to Market Browser

  4. TOP STORIES (news_events[1:5])
     - 2-column grid layout
     - Each card: section label, headline, byline, article body, odds tags
     - Multi-outcome events show grid of outcome cards
     - "Explore" button on each card

  5. MARKET WATCH (news_events[5:10])
     - 3-column compact grid
     - Each card: title, top 3 outcomes with probabilities, 24h volume
     - "Details" button on each card

  6. HOT TAKES (news_events[10:])
     - Purple-accented opinion cards
     - Left border accent, italic text
     - "Explore" button on each card

  7. FOOTER
     - AI mode indicator (Gemini vs template)
     - Disclaimer about odds not being guarantees

NEWSPAPER HELPER FUNCTIONS:
  _parse_market(mkt)          -> (outcomes, prices, top_outcome, top_prob)
  _is_binary_yesno(outcomes)  -> bool (True if outcomes are exactly Yes/No)
  _is_multi_outcome(ev)       -> bool (True if named outcomes or multiple sub-markets)
  _get_all_outcomes(ev)       -> [(name, prob), ...] sorted by prob desc
  _odds_tags_html(...)        -> HTML string of tag pills
  _multi_outcomes_html(...)   -> HTML string of outcome card grid
  _tag_str(ev)                -> "POLITICS · ELECTIONS" label from event tags
  _reading_time(text)         -> int (words / 200, min 1)
  _add_icon(name)             -> "🇺🇸 United States" / "⚽ Barcelona" — prepends entity icon
  _polymarket_link_html(slug, label) -> Styled HTML link to polymarket.com/event/{slug}
  _render_lead(ev, text)      -> Renders lead story HTML
  _render_article(ev, text)   -> Renders standard article HTML
  _render_compact(ev)         -> Renders compact market watch card
  _render_opinion(ev, text)   -> Renders hot take card

  First option "All Analytics" -> analytics mode

    - Multi-outcome grid (if applicable)
    - Expandable market panels with bet calculator or resolution info
    - Price history chart for first market
    - "Back to The Chronicle" and "Dismiss" buttons

CATEGORY BROWSER:
  1. CATEGORY LOADING
     get_top_categories(limit=14) scans top 500 events by volume,
     counts tag frequencies, returns [{label, slug, count}]
     Cached 5 min via @st.cache_data(ttl=300)

  2. KPI ROW (5 columns)
     Events count, Markets count, Total Volume, Active count, Resolved count

  3. FILTER CONTROLS
     - Text search (filters by event title and market question)
     - Status filter: All / Active / Resolved
     - Sort: Volume desc/asc, Newest, Oldest

  4. EVENT CARDS (paginated, 15 per page)
     Each event renders:
       - HTML card with title, status tag, volume tag, market count
       - Multi-outcome badge ("N outcomes") if applicable
       - Event tags from Gamma API
       - "View on Polymarket" link (opens polymarket.com/event/{slug} in new tab)

  5. MULTI-OUTCOME OVERVIEW EXPANDER
     For events with multiple sub-markets:
       - Horizontal bar chart showing all candidates and probabilities
       - Color-coded bars from palette: blue, green, red, yellow, purple...
       - Caption about what probabilities represent

  6. INDIVIDUAL MARKET EXPANDERS
     For each market (up to 15 per event), sorted active-first then by volume:

     ACTIVE MARKET -> Bet Payout Calculator:
       - Selectbox: pick outcome (Yes/No or named outcomes)
       - Number input: bet amount ($1 - $100K, default $100)
       - Calculated results:
           Payout = bet / price_of_chosen
           Profit = payout - bet
           ROI = profit / bet * 100
           Implied prob = price
           Odds = 1 / price
       - Volume and outcomes caption
       - "Place this bet on Polymarket" link (opens real Polymarket event page)

     RESOLVED MARKET -> Detail Panel:
       - Winner metric, Volume metric, Duration metric
       - "Final Resolution" list for non-binary markets
       - P&L estimates (binary markets only):
           Uses lastTradePrice as proxy for crowd confidence
           Winners gain ~ lose_prob * volume
           Losers lose  ~ lose_prob * volume
       - Market efficiency label (green/yellow/red)
       - Price history chart:
           Fetches from CLOB Prices History API
           Shows outcome line(s) with fill-to-zero for binary
           Multi-outcome: up to 4 colored lines with legend
           Resolution line for binary (green at 1.0 or red at 0.0)
       - "View on Polymarket" link at bottom of resolved panel

  POLYMARKET LINKS:
     Built by _polymarket_link_html(slug, label) helper.
     URL pattern: https://polymarket.com/event/{slug}
     The slug comes from ev.get("slug") on the event object (Gamma API).
     Styled as a blue bordered button with hover effect, opens in new tab.
     Placed at: event card header, active market expander, resolved market expander,
     and featured market panel.

  7. PAGINATION
     "Load More Events" button adds 15 more (st.session_state.cat_limit)

6. MODE 3: ALL ANALYTICS
================================================================================

Location in app.py: Lines ~1920-end (~2278)

DATA SOURCE:
  load_markets() reads data/raw/markets_raw.json (static offline dataset)
  Returns pd.DataFrame with derived columns:
    - primary_tag: category label (heuristic mapping from event tags/category)
    - resolved_winner: outcome with price >= 0.99, else "Unresolved"
    - implied_prob: lastTradePrice as float (0-1)
    - is_binary_yesno: True if outcomes are exactly {"yes", "no"}
    - crowd_win_pct / crowd_lose_pct: winner/loser share of volume
    - est_loser_loss / est_winner_gain: estimated P&L from probabilities
    - crowd_was_right: True if crowd_win_pct >= 0.5
    - duration_days: endDate - startDate in days
    - end_month: year-month period string for timeline

P&L COMPUTATION (compute_pnl):
  Binary markets: win_prob = prob if winner is outcomes[0], else 1-prob
  Non-binary markets: win_prob = prob directly (lastTradePrice of winner's share)
  lose_prob = 1 - win_prob

EFFICIENCY COMPUTATION (get_efficiency):
  For resolved markets: efficiency = implied_prob * 100
  Handles binary and non-binary the same way (uses prob directly)
  >= 70% -> high confidence
  >= 50% -> divided
  < 50%  -> upset (crowd got it wrong)

SIDEBAR FILTERS:
  - Search by question text
  - Winner filter: specific outcome
  - Accuracy filter: Accurate / Inaccurate / All
  - Multi-select tags filter
  - Volume range slider
  - Date range filter

CHARTS:
  1. Top 15 Markets by Volume (horizontal bar chart)
     Colors: green for "Yes" winners, red for "No", purple for named winners
  2. Volume by Category (treemap)
  3. Markets Closed per Month (timeline)
  4. Market Accuracy by Category (bar chart, shows % crowd-was-right)
  5. Biggest Upsets (where crowd got it wrong, crowd_was_right=False)
  6. Market Efficiency Distribution (histogram of efficiency scores)

MARKET CARDS:
  For each filtered market (paginated):
    - Question, winner (green badge or red badge), probability
    - Volume, duration, accuracy stats
    - P&L row: winners gain / losers loss estimates

7. MULTI-OUTCOME MARKET DETECTION
================================================================================

The Polymarket API has two types of non-binary markets:

TYPE 1: Single market with multiple named outcomes
  Example: "Who will win the election?" -> outcomes: ["Trump", "Biden", "DeSantis"]
  Detection: outcomes list is NOT {"yes", "no"}

TYPE 2: Multi-market events (each sub-market = one candidate)
  Example: Event "Who wins the election?"
    Market 1: "Trump?" -> Yes 65%, No 35%
    Market 2: "Biden?" -> Yes 25%, No 75%
    Market 3: "DeSantis?" -> Yes 8%, No 92%
  Detection: len(markets) > 1

Helper functions:

  _is_binary_yesno(outcomes: list) -> bool
    Returns True ONLY if outcomes == {"yes", "no"} (case-insensitive, exactly 2)

  _is_multi_outcome(ev: dict) -> bool
    Returns True if:
      - First market's outcomes are NOT Yes/No, OR
      - Event has multiple sub-markets (len(markets) > 1)

  _get_all_outcomes(ev: dict) -> list[tuple[str, float]]
    Returns sorted [(name, probability), ...] descending by prob
    For Type 1: directly uses outcomes and prices from market
    For Type 2: extracts "Yes" price from each sub-market's question
    For standard binary: returns [(outcome, price), ...]

These helpers are used in:
  - Newspaper rendering (outcome grids, ticker, odds tags)
  - Market Browser (outcome overview panel, market badges)
  - LLM prompt building (market type annotation)
  - Analytics (P&L computation, efficiency, chart colors)

10. LLM ARTICLE GENERATION SYSTEM (gemini_writer.py)
================================================================================

OVERVIEW:
  Single batch API call to Gemini gemini-2.5-flash model.
  All 15 newspaper events sent in one prompt.
  Returns JSON mapping event_id -> article text.
  Falls back to template articles on failure.

ARTICLE STYLES (5 total, assigned deterministically via MD5 hash of event ID):
  1. breaking_news  - urgent, AP-style wire reporting, inverted pyramid
  2. analysis       - measured, Financial Times column, analyze numbers
  3. market_watch   - concise, Bloomberg terminal style, data-first
  4. opinion        - provocative, contrarian, editorial page voice
  5. human_interest - narrative, New Yorker storytelling

SYSTEM PROMPT (WRITING PROTOCOL):
  Voice: "sharp, confident, slightly irreverent -- like Matt Levine meets Bloomberg"
  8 rules:
    1. LENGTH: 3-5 sentences, 60-100 words, no filler
    2. OPENER: Lead with most interesting number/contrast/claim
       Banned openings: "A new market...", "Traders are watching...", etc.
    3. DATA WEAVING: "a commanding 81%" not "81%", contextualize all numbers
    4. VOICE: Active only, strong verbs (surges, collapses, defies, commands)
    5. CLOSER: End with catalyst or tension, make reader want to check back
    6. FORMAT: Plain prose, no markdown/bullets/emojis
    7. MULTI-OUTCOME: Horse-race narrative, name frontrunner and dark horse
    8. VARIETY: Each article must feel distinct, vary openers

BATCH PROMPT BUILDING (_build_batch_prompt):
  For each event, includes:
    - Market ID, Title, Question
    - Current Odds (JSON dict)
    - 24h Volume, Total Volume
    - Tags
    - Market Type annotation (MULTI-OUTCOME with candidate count if applicable)
    - Assigned style name + tone + structure

MARKET SUMMARY BUILDING (_build_market_summary):
  Detects market_type:
    - "binary": standard Yes/No
    - "multi_outcome": single market with named outcomes
    - "multi_market": event with multiple sub-markets (each = candidate)
  For multi_market: aggregates all sub-markets' "Yes" prices into unified odds dict

RESPONSE PARSING (_parse_batch_response):
  1. Strip markdown code fences if present
  2. Parse JSON dict
  3. Validate: each article must be string with length > 20
  4. Fill any missing with _fallback_article()

FALLBACK TEMPLATES (when no API key or API error):
  Binary: _OPENERS templates with volume/probability/strength variables
  Multi-outcome: _MULTI_OPENERS templates with frontrunner/runner-up/n-way
  _CLOSERS: 5 forward-looking sentence templates
  Random selection seeded by event title (deterministic per event)

API CONFIGURATION:
  Model: gemini-2.5-flash
  Temperature: 0.85
  Max output tokens: 4000
  Library: google-genai (google.genai + google.genai.types)

11. API DATA LAYER (fetch_events.py)
================================================================================

FUNCTIONS:

fetch_events_by_category(tag_slug, max_pages=5, include_closed=True)
  Fetches paginated events from Gamma API, filtered by category tag.
  100 events per page, up to max_pages pages.
  Retry logic: 3 attempts with exponential backoff (2^attempt seconds).
  Deduplicates by event ID.
  Used by: Market Browser mode.

get_top_categories(limit=15)
  Scans top 500 events (by volume) from Gamma API.
  Counts tag frequencies across all events.
  Normalizes slugs: "trump-presidency" -> "trump", "us-current-affairs" -> "us-politics"
  Filters out "potusbanner" tags.
  Returns: [{label, slug, count}] sorted by count.
  Used by: Market Browser category pills.

fetch_price_history(token_id, fidelity=60)
  Fetches historical price data from CLOB Prices API.
  Returns: [{t: unix_timestamp, p: price_float}]
  Used by: Market Browser price charts, Featured Market panel.

fetch_newspaper_events(limit=15)
  Fetches active, high-interest events for the newspaper.
  Query params: order=volume24hr, closed=false, active=true
  Fetches 4x the limit, then scores and filters.
  Score = (vol_24h * time_score) + (competitive * 100000)
  Used by: Newspaper mode.
